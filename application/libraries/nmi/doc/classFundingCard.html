◆ getCardHash()

FundingCard::getCardHash ( )

Gets the card hash found in the Funding Card within the response that can be used to reference a card in a follow-up transaction.

This is an alphanumeric string with a fixed length of 28 characters. Used in conjunction with the CardReference property. The benefit of being able to reference a previously used card is that an integrator need not store actual card details on their system for repeat transactions. This reduces the risk of card information being compromised, and reduces the integrators PCI requirements.

Returns
string The card hash found in the response that can be used to reference a card in a follow-up transaction.

◆ getExpiryDateFormat()

FundingCard::getExpiryDateFormat ( )

Gets the expiry date format associated with the card found in the Funding Card within the response.

This will match the format of the expiry date and can include separators such as - and /. This is a character string with a maximum length of 10 characters. The available options are shown in the following table:

Format Description Example
yyyy Year with century 2004
yy Year without century 04
MM Month of year 01
dd Day of month 27
Returns
string The expiry date format associated with the card in this response. This will be null if no expiry date format was found in the response.

◆ getPan()

FundingCard::getPan ( )

Gets the masked PAN (Primary Account Number) found in the Funding Card within the response.

The PAN is masked with x's for security. This is an alphanumeric string with a minimum length of 12 characters and a maximum length of 19 characters.

Returns
string The PAN (Primary Account Number) found in this response. This will be null if no PAN was found in this response.
